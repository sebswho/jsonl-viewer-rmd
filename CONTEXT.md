# 项目上下文

## 术语表

- 记录（Record）：左侧列表中的一条可选数据项。JSONL 中一行对应一条记录；JSON 中顶层数组的每个元素对应一条记录；非数组顶层值整体对应一条记录。
- 文件页签（File Tab）：一个已打开文件在界面中的独立工作上下文，包含文件名、记录集合、过滤词、排序方向等状态。
- 可恢复文件（Restorable File）：通过 Chromium `showOpenFilePicker()` 打开的文件，应用可持久化其文件句柄和界面状态，在刷新或重新打开页面后恢复。
- 临时文件（Ephemeral File）：通过拖拽或普通文件输入打开的文件，只在当前页面会话中可见，不保证跨会话恢复。
- 会话恢复（Session Restore）：浏览器重新加载页面后，恢复上一次未关闭的可恢复文件页签及约定的界面状态。
- 记住会话（Remember Session）：控制是否把会话恢复元数据持久化到浏览器本地存储的用户开关。

## 当前系统边界

- 技术栈：Next.js App Router + React 19 + TypeScript。
- 当前主流程集中在 `src/app/page.tsx` 单文件中。
- 当前应用已支持：
  - JSONL 文件按行解析和展示
  - 多标签页
  - 过滤、排序、原始/格式化/树形视图
  - 行大小显示开关（已使用 `localStorage` 持久化）

## 本次变更新增约束

- 默认交互语言改为中文，不引入多语言体系。
- `.json` 和 `.jsonl` 共用同一套查看体验。
- 仅承诺 Chromium 浏览器的文件恢复能力。
- `temp/` 仅用于本地调试文件，不纳入版本控制。
